### 좀비 프로세스 - 종료 후 방치된 자식 프로세스

자식프로세스

프로세스가 종료될 때 PCB에 종료코드를 저장한다 회수 될 때 까지.

PCB와 프로세스 테이블의 항목은 제거되지 않음 → PID

1. shell
    
    바로 wait()를 호출
    
2. 서버
    
    wait()를 쓰면 프로세스 생성과 같은 다른 클라이언트 작업이 모두 정지되서 사용 불가능. 
    
    child를 생성하고 wait()하고 있지 않고 다른 child를 생성하고 있기에 SIGCHLD(~~ child death) 신호가 필요하다. 
    

좀비 프로세스를 제거 방법

1. 쉘에서 부모 프로세스에게 신호를 보내 종료시키게 만듬. 종료안시키면 아래 방법.
2. 부모 프로세스 강제 종료 - 좀비 고아 발생 → init에게 상속(입양)→init에 의해 종료

좀비 프로세스는 메모리가 제거 되어 사용량이 N/A라고 표기되거나 defunc로 나타남.

고아 프로세스(orphan process) - 종료 되었을 때도 되기도함(회수 아된 상태, 좀비).

유닉스는 일일이 자식 프로세스를 제거해야한다.
