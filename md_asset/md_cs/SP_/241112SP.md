프로세스 오버레이, exec()

PID 변경 없음, 기존의 프로그램위에 덮어씌워 실행

프로세스가 실행할 프로그램의 내용이 바뀌는 것.

새로운 실행 파일을 적재하여 기존의 것을 덮어쓴다 → exec()을 호출한 프로그램이 사라진다.

새로운 프로세스의 생성이 아니다, 유닉스에서 프로세스 생성은 fork()다.

fork()의 두가지 종류 

1. 자기와 똑같은 process(network server) - 자식 프로세스를 생성하여 클라이언트에 서비스 제공
2. shell(login할 때 만들어진 shell , login 쉘 → 사라지면 안된다.) : 사용자 입력 명령어 실행. 사라지면 안되기에 fork를 하여 자식 프로세스에서 exec()을 호출한다 이것이 ppt에서 설명된 경우이다.

vfork(): copy on write를 하게되면 부모에게 영향을 주지 않는다 하지만 vfork()는 자식이 그냥 부모 자원 사용 → 부모에게 영향을 줌. 

exec 패밀리 시스템 콜은 6개가 있다.  이중 execve()만이 시스템 콜이 나머지는 라이브러리 함수

인자 전달이 어렵다(위한 준비가 포인터를 굉장히 많이 사용한다고함.) 그래서 나머지는 인자 전달을 쉽게하기 위한 라이브러리 함수라고 생각하자.

exec l, p , v, e (관련 ppt 따로 만들어주신다고함)

l = 명령어 인자가 list 형태로 온다는 것 linked list가 아닌 나열해서 준다는 뜻.

v = 명령어 인자가 vector 형태로 온다는 것 l과는 같이 사용될 수 없다.

e =  환경 변수(vector 형태)를 새로 주겠다는 의미

p = path를 표시

1. 실행 파일
2. 명령어 입력형태 (c언어 에서 argument vector)
3. 환경 변수(vector) 개수가 많아 질 수 있다. 그렇게되면 인자 처리가 어려워지기에 라이브러리(exec 패밀리)가 있다. 

main 인자로 받는 것처럼 “”단위로 인자를 하나씩 받아 pointer로 하나씩 연결해 vector Array 형태로 만들어준다음 보내줘야한다.

### 프로세스 종료

exit() , _exit()

_가 붙어있는 경우는 시스템에 가깝다는 뜻으로 사용자가 써도 되지만 내부적으로 사용하겠다는 의미다.

각 레벨마다 _가 추가됨.

program : func_del()

c: _*func_*del()

kernel : __*func_*del()

exit() - c lib의 존재 인지 → buffer가 있다(buffer 정리 필요), 라이브러리에 가깝다.,

_exit()는 다날림.

종료는 자동으로 해주지만 우리가 안해버릇하면 일이 중첩되었을 때 exit() 누락될 수있다.

종료코드 부모에게 결과를 알려주는 코드

부모 프로세스가 wait()으로 시간을 맞추는 방식(sychronous) - block 당할 수도 있다. 기다리는 경우가 많다.

비동기적 asynchrous 시그널을 받을 때 알려주는 방식 자식이 좀비 상태로 대기하는경우.

종료코드는 종료한 상태나 이유를 알려주는 것.

0이 정상, 1~255 개발자가 종료 이유를 임의로 저장

255이상은 사용에 유의해야한다.
